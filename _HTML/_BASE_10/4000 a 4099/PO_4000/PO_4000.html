• INDEXES OU INDICES

Índice (index) é uma estrutura especial que melhora a eficiência das consultas (queries) em coleções. Ele funciona de forma semelhante ao índice de um livro, permitindo que o banco de dados encontre os documentos mais rapidamente, sem precisar escanear toda a coleção.

Por que usar índices no MongoDB?

Sem índices, o MongoDB precisa fazer um collection scan, ou seja, verificar cada documento da coleção para encontrar os que correspondem à consulta. Isso pode ser muito lento em coleções grandes.Com índices, o MongoDB pode pular diretamente para os documentos relevantes

Tipos de índices no MongoDB

  • 1. Índice padrão (_id)
    • • Toda coleção tem automaticamente um índice no campo _id.
  • 2. Índice de campo único
    • • Indexa um único campo.
    • • Exemplo: db.usuarios.createIndex({ nome: 1 }) // 1 = ordem crescente
  • 3. Índice composto
    • • Indexa múltiplos campos.
    • • Exemplo: db.usuarios.createIndex({ nome: 1, idade: -1 }) // 1 = ordem crescente, -1 = ordem decrescente
  • 4. Índice único (unique)
    • • Garante que os valores do campo sejam únicos.
    • • Exemplo: db.usuarios.createIndex({ email: 1 }, { unique: true })
  • 5. Índice geoespacial
    • • Para consultas com localização (latitude/longitude).
    • • Exemplo: db.usuarios.createIndex({ localização: "2dsphere" })
  • 6. Índice de texto (text)
    • • Para buscas em campos de texto.
    • • Exemplo: db.artigos.createIndex({ conteúdo: "text" })

Cuidados ao usar índices

  • • Os índices ocupam espaço em disco e podem impactar a performance de gravações.
  • • É importante monitorar o uso de índices e removê-los quando não forem mais necessários.
  • • Considere o uso de índices apenas quando necessário e evite indexar campos com alta cardinalidade.
  • • Use o comando explain() para entender como suas consultas estão utilizando os índices.
  • • Não esqueça de testar suas consultas com e sem índices para avaliar o impacto na performance.
  • • Considere o uso de índices apenas quando necessário e evite indexar campos com alta cardinalidade.
  • • Muitos índices podem degradar a performance de escrita (inserções, atualizações).
  • • Use índices em campos que são frequentemente consultados.
  • • É importante analisar as queries com explain() para saber se estão usando os índices corretamente.
